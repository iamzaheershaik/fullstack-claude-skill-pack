Backend runtime
Runtime
Node.js + Express / Fastify You know this Non-blocking I/O. Excellent for API-heavy, real-time apps. Fastify is 2x faster than Express — switch immediately for production.
Java 17+ / Spring Boot 3 MNC standard Dominant in Indian IT services. Typed, verbose, but battle-tested at massive scale. Spring ecosystem is enormous.
Go (Golang) / Python FastAPI Interview differentiator Go for ultra-low latency microservices (Uber, Cloudflare). FastAPI for ML-adjacent APIs. Go compiles to single binary — zero runtime overhead.
Frontend
Frontend
React + Next.js You know this SSR/SSG, App Router, RSC. Industry default for product companies. Next.js 14+ App Router is the current production standard.
Angular 17+ Enterprise default Opinionated, full framework. Strongly typed with TypeScript by default. Preferred in Indian banking, insurance, and large enterprise IT.
Vue 3 / Nuxt / SvelteKit Growing Vue dominant in China-adjacent companies. SvelteKit compiles away the framework — zero runtime bundle. Real differentiator for performance-focused roles.
Relational database (OLTP)
SQL DB
PostgreSQL 15+ Best default JSON support, PostGIS, full-text search, LISTEN/NOTIFY, logical replication. The correct answer for 90% of OLTP workloads. Master this.
Oracle DB / MySQL Legacy MNC Oracle: banking, telecom, government contracts. MySQL: legacy web apps, WordPress hosts. Both are being slowly replaced by PostgreSQL in greenfield projects.
CockroachDB / PlanetScale / Spanner Distributed SQL Horizontally scalable SQL. Spanner is Google's internal database. CockroachDB is Postgres-compatible. Use when sharding PostgreSQL becomes painful.
NoSQL / Document store
NoSQL
MongoDB Atlas You know this Document model. Right for flexible schemas, time-series, content. Wrong for financial transactions and complex joins. Don't use it as a PostgreSQL replacement.
Cassandra / DynamoDB Massive scale Leaderless replication. Write-heavy, high cardinality. Netflix, Instagram, Discord use Cassandra. DynamoDB is AWS-managed. Zero joins — design schema around queries.
Firestore / Bigtable / HBase Specialized Firestore: real-time sync for mobile apps. Bigtable/HBase: petabyte-scale analytics, IoT telemetry. Hadoop ecosystem but being replaced by cloud-native.
Cache layer
Cache
Redis 7 (ElastiCache) Industry standard Strings, hashes, sorted sets, pub/sub, streams. Master all 5 data structures — each has different interview relevance. Cluster mode for horizontal scale.
Memcached / Hazelcast Niche Memcached: simpler, multi-threaded, no persistence. Hazelcast: in-memory data grid for Java ecosystem, distributed computing. Mostly encountered in legacy Java shops.
Dragonfly / KeyDB Emerging Redis-compatible, multi-threaded (Redis is single-threaded per shard). 25x throughput claims. Too new for most enterprise but interview-worthy to know about.
Message queue / event streaming
Queue
RabbitMQ / BullMQ Node.js native RabbitMQ: complex routing, work queues, fanout. BullMQ: Redis-backed job queues for Node.js. Use for task processing; NOT a good fit for event sourcing.
Apache Kafka (MSK) System design essential Append-only log. Retained, replayable, partitioned. The correct answer for event sourcing, CDC, audit logs, and high-throughput pipelines. Know this deeply for MNC interviews.
Pulsar / Kinesis / NATS Alternatives Pulsar: Kafka alternative with multi-tenancy, tiered storage. Kinesis: AWS-managed, less flexible. NATS: ultra-fast, lightweight, no persistence — for internal microservice comms.
Search engine
Search
Elasticsearch / OpenSearch Industry default Full-text + geo + aggregations. OpenSearch is AWS's open-source fork. Know inverted index internals — frequently asked in senior design interviews.
Solr / Lucene Legacy Elasticsearch is built on Lucene. Solr is also Lucene-based. You'll find Solr in legacy Java enterprise. Not worth learning from scratch — learn ES instead.
Typesense / Meilisearch / Algolia Modern alternatives Typesense/Meili: simpler ops, instant search, typo-tolerance. Algolia: SaaS, expensive, dead simple. All three replace ES for product search at smaller scale. Algolia is interview-safe to mention.
Cloud infrastructure
Cloud
AWS (primary) / GCP Standard AWS dominates market share. GCP is preferred for ML/data workloads (BigQuery, Vertex AI). Know core AWS: EC2, S3, RDS, EKS, Lambda, CloudFront, VPC, IAM.
Azure MNC / enterprise Dominant in Indian IT services (TCS, Infosys clients are often on Azure due to Microsoft enterprise agreements). AKS for Kubernetes, Cosmos DB, Azure Service Bus.
GCP / Multi-cloud ML-heavy orgs GCP: BigQuery (fastest OLAP), Vertex AI, Spanner. Multi-cloud is expensive operationally — most companies are single-cloud. Mention it cautiously in interviews.
Container orchestration
Orchestration
Docker + Kubernetes (EKS/GKE) Non-negotiable You must know Kubernetes at a conceptual level for any senior role. Pods, Deployments, Services, Ingress, HPA, ConfigMaps, Secrets. EKS is the AWS managed flavor.
AWS ECS / Fargate Simpler K8s alt ECS: AWS-native container orchestration. Fargate removes server management. Easier than Kubernetes but less portable. Common in AWS-first small teams.
Nomad / OpenShift Niche Nomad (HashiCorp): simpler than K8s, handles non-container workloads. OpenShift: Red Hat's K8s distribution, common in on-premise enterprise. Not interview-critical.
Observability
Observability
Datadog / New Relic SaaS standard Metrics, traces, logs in one platform. Expensive but zero ops overhead. Datadog dominates product companies. Know APM, distributed tracing, and dashboard creation.
Prometheus + Grafana + Jaeger Open source standard Prometheus for metrics (pull-based, PromQL). Grafana for dashboards. Jaeger/Tempo for distributed tracing. This combo is the interview-safe answer for any system design question about observability.
ELK Stack / OpenTelemetry Logging layer ELK (Elasticsearch + Logstash + Kibana): structured log aggregation. OpenTelemetry: vendor-neutral instrumentation standard — instrument once, export to any backend. Learn OTel — it's becoming the standard.
CI/CD pipeline
CI/CD
GitHub Actions + ArgoCD Modern standard GitHub Actions for CI (build, test, Docker push). ArgoCD for GitOps-based CD to Kubernetes. This is the current gold standard for product companies.
Jenkins / Azure DevOps Enterprise / MNC Jenkins: self-hosted, highly configurable, heavily used in Indian IT services. Azure DevOps: end-to-end Microsoft stack. Both are declining in greenfield but dominant in legacy enterprise.
GitLab CI / CircleCI / Tekton Alternatives GitLab CI: strong if company already uses GitLab. CircleCI: fast, simple YAML config. Tekton: Kubernetes-native pipelines. Tekton knowledge is rare and impressive in interviews.
API layer
API style
REST + OpenAPI / GraphQL You know this REST for external APIs. GraphQL for mobile/multi-client with varied data needs. OpenAPI (Swagger) for documentation and contract-first design. Standard interview territory.
gRPC + Protocol Buffers Internal microservices Binary protocol, strongly typed, bidirectional streaming. 7-10x faster than REST for internal service communication. Used by Google, Netflix, Uber. Critical to understand for senior system design.
tRPC / TRPC + WebSockets TypeScript-first tRPC: end-to-end type safety, no codegen, Next.js native. WebSockets for real-time. You already know WS from BakeRoute. tRPC is worth knowing for full-stack TypeScript roles.
Analytics / data warehouse
Analytics
ClickHouse Best open-source OLAP Column-oriented. Fastest analytical queries at scale. Used by Cloudflare, Uber, ByteDance. Never run analytics on your PostgreSQL OLTP — this is the answer to "where do you put it instead."
BigQuery / Redshift / Snowflake Cloud OLAP BigQuery: serverless, pay-per-query, fastest to onboard. Redshift: AWS-native, columnar. Snowflake: multi-cloud, popular in US enterprises. All are correct answers in interviews; pick by cloud alignment.
Apache Spark / Flink / dbt Data engineering Spark: batch processing at petabyte scale. Flink: real-time stream processing. dbt: SQL-based transformation layer sitting on top of a warehouse. Data engineering track, not backend track.
